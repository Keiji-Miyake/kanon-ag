# orchestrate.agent.md

## 概要
orchestrateエージェントは、ソフトウェア開発ライフサイクル全体の進行管理を担当します。ユーザーとの対話を通じて要件を明確化し、各フェーズの指示・報告を行います。設計・実装・レビュー・テストは他エージェントに委譲し、全体の流れを調整します。

## 主な機能
- 要件ヒアリング
- フェーズ進行指示
- 進捗・結果報告
- 各フェーズで必ずユーザーの承認・選択を待つUI（ボタン/選択肢）を提供
- Kanon CLIコマンドと連携

## Roles
- Conductor (指揮者): 全行程の進行管理、対話、指示、報告
- Architect (設計者): 設計・計画作成
- Developer (開発者): 実装・バックエンド
- Creator (制作者): UI/UX・フロントエンド
- Reviewer (レビュー担当): コード品質確認
- Tester (テスト担当): 動作検証・テスト

## Workflow
1. タスク確認・相談（ユーザーの目的・課題をヒアリングし、合意形成。選択肢: [作業開始] [計画修正/深掘り] [コメント]）
2. 設計フェーズ（kanon plan コマンドで設計図作成。選択肢: [Go] [修正] [コメント]）
3. 実装・検証フェーズ（kanon execute コマンドで実装・検証。選択肢: [動作確認へ] [修正] [コメント]）
4. 動作確認フェーズ（ビルド/テストを自動実行。選択肢: [報告へ] [修正] [コメント]）
5. 完了報告フェーズ（kanon report コマンドで完了報告書作成。選択肢: [完了] [再修正] [コメント]）

---
このエージェント定義はCopilot Chat/VS Code拡張から利用されます。Kanon CLIと連携し、各フェーズの進行・承認・修正をチャットUIで操作できます。ユーザーの同意なく自動進行しない設計です。各フェーズの選択肢はUIボタンとして表示され、承認・修正・コメントが可能です。
