# 概要
Kanon のエージェント定義を、再利用可能な 4つの要素（Persona, Policy, Knowledge, Instruction）に分解し、これらを動的に組み合わせてプロンプトを構築する **Faceted Prompting (プロンプトのモジュール化)** を導入する。

# 機能要件
1. **ファセット・モデルの定義**:
   - `Persona`: エージェントの性格、専門性。
   - `Policy`: 遵守すべき規約、ガイドライン（例: コーディング規約）。
   - `Knowledge`: プロジェクト固有の知識、アーキテクチャ。
   - `Instruction`: 特定のタスクに対する具体的な指示。
2. **ファイル管理構造の導入**:
   - `facets/` ディレクトリ配下に、各要素を Markdown ファイルとして配置。
3. **PromptAssembler の実装**:
   - 指定されたファセット・ファイルを読み込み、単一のシステムプロンプトとして結合するエンジン。
4. **Passage との統合**:
   - `Passage` 定義で、特定のファセットを指定可能にする。
   - 例: `persona: "planner", policies: ["coding-standard"], knowledge: ["architecture"]`

# 非機能要件
- 既存の固定的なプロンプト生成ロジックとの互換性を保ちつつ、新しい仕組みへの移行をサポートする。
- TDD に従い、プロンプト結合ロジックのテストを先行して実装すること。

# 受け入れ基準
- `Passage` で指定されたファセットが正しく結合され、エージェントに送信されること。
- ファセットの組み合わせにより、一貫した出力を得られること。
