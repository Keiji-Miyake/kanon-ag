# 概要
複数のエージェントを並列実行し、その結果を LLM（Supervisor）が統合・判定する **Consensus (合議制/MAGI方式)** を Kanon の Score エンジンに導入する。結論が不一致の場合は、相互のフィードバックを伴う再審議ループを実行する。

# 機能要件
1. **Passage の並列実行サポート**:
   - `Passage` モデルを拡張し、単一の `skill` だけでなく `skills: string[]` による複数エージェント指定を可能にする。
   - `ScoreExecutor` または `orchestrate.ts` でこれらを並列に Spawner で起動し、出力を収集する。
2. **ConsensusService (LLM集計) の実装**:
   - 収集した複数のエージェント出力を一つのプロンプトにまとめ、`supervisor` スキルを持つエージェントに判定（`json:passage-result` の生成）を依頼する。
3. **再審議ループ (Deliberation Loop) の導入**:
   - `supervisor` が結論を出せなかった（または再審議を求めた）場合、収集した全出力を各エージェントへの「背景知識（Knowledge）」として渡し、2回目の実行を行う。
   - 最大ループ回数を設定し、解決しない場合は ABORT する。

# 非機能要件
- 並列実行による API コストとトークン消費の増加を考慮し、デバッグモード等で実行を制限可能にすること。
- TDD に従い、集計ロジック（複数出力からのプロンプト構築）のテストを先行すること。
