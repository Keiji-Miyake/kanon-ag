# 概要
Kanon のオーケストレーターが、各 Score 実行に対して専用の `git worktree` を自動的に作成・管理・統合する機能を導入する。これにより、エージェントの作業環境をメインブランチから隔離し、安全かつ並列な開発を可能にする。

# 機能要件
1. **自動ワークツリー作成 (Score-level)**:
   - `kanon score` 開始時に、ランダムまたはタスク名に基づくユニークなブランチ名で `git worktree add` を実行する。
   - ワークツリーの作成先（ベースディレクトリ）は設定ファイル（`kanon-config.json`）でカスタマイズ可能にする（デフォルトはプロジェクトルートの `worktree/`）。
2. **コンテキストの自動切り替え**:
   - ワークツリー作成後、その Score 内の全 Passage は自動的にそのワークツリーのディレクトリを `cwd`（カレントディレクトリ）としてエージェントを起動する。
3. **成果物の自動マージ & クリーンアップ**:
   - Score が成功（`next_passage` が `null` で正常終了）した場合、そのブランチを自動的に元のブランチ（`main` 等）にマージし、ワークツリーを削除する。
   - マージ衝突が発生した場合は ABORT し、人間に解決を委ねる。
4. **環境変数の伝播**:
   - ワークツリーのパスを `KANON_CURRENT_WORKTREE` などの環境変数としてエージェントに提供する。

# 非機能要件
- `LocalGitSandbox` クラスを再利用・拡張し、信頼性の高い Git 操作を実現すること。
- TDD に従い、ワークツリーのライフサイクル管理のテストを先行すること。
