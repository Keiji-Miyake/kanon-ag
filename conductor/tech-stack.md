# 技術スタック

このドキュメントでは、Kanonプロジェクトで使用される主要な技術とアーキテクチャパターンについて説明します。

## 主要言語

- **TypeScript (Node.js):** CLIとVS Code拡張機能の両方の主要言語であり、コードベース全体の型安全性と一貫性を確保します。

## フレームワーク＆ライブラリ

- **Node.js (CLI):** オーケストレーションエンジンの実行環境。
- **VS Code Extension API:** VS Codeダッシュボードの構築と開発者のIDEとの統合に使用されます。
- **Vitest:** ユニットテストおよび統合テスト用の主要なテストフレームワーク。
- **WebSocket (WS):** CLI（サーバー）とVS Codeダッシュボード（クライアント）間のリアルタイム通信を可能にします。
- **Chalk:** スタイル化および色付けされたCLI出力に使用されます。
- **YAML & Gray-matter:** 設定ファイルやMarkdownファイルのフロントマターを解析します。

## データストレージと状態管理

- **Redis:** エージェント間の通信用の共有黒板（Blackboard）および複雑なオーケストレーションタスク中の永続的な状態管理に使用されます。
- **In-Memory Blackboard:** ローカルまたは開発環境向けのRedisの軽量な代替手段。
- **有限状態マシン (FSM):** エージェントの状態遷移とタスクの進行を管理するためのコアメカニズム。

## アーキテクチャとデザインパターン

- **クリーンアーキテクチャ:** プロジェクトをドメイン、ユースケース、インフラストラクチャ、およびインターフェースレイヤーに分割することで、関心の分離を確実にします。
- **ドメイン駆動設計 (DDD):** 実装をコアビジネスロジック（オーケストレーション、エージェントの状態、フィードバック）に集中させます。
- **マルチエージェント・オーケストレーション:** 専門のエージェントが順次または並行して協力し、共通の目標を達成するパターン。

## ツールとインフラストラクチャ

- **Git:** プロジェクト管理とタスク固有のワークツリーに使用されるバージョン管理システム。
- **プロセスグループ管理:** 子プロセスとその子孫を確実にクリーンアップするために `detached: true` とマイナスPIDシグナルを利用。
- **npm:** 依存関係管理とビルドスクリプト用のパッケージマネージャー。
- **JSON Schema:** 設定ファイル（`kanon-cli.json`）のIDE補完とバリデーションを提供するために使用されます。
