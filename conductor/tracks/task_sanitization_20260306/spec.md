# 仕様書: WorktreeManagerの堅牢性のためのタスク名サニタイズ

- **目標:** `WorktreeManager` を強化し、Gitブランチやワークツリーを作成する前に、長いタスク名や複雑なタスク名を堅牢にサニタイズおよび切り詰め（Truncate）できるようにします。
- **課題:** 現在の `WorktreeManager.ts` の `safeTaskId` ロジックは文字の置換のみを行っており、長い文字列の切り詰めを行っていません。極端に長いタスク名は、Gitのブランチ名制限（通常は約255文字ですが、ファイルシステムの関係でより短い方が安全です）を超えたり、置換後に重複したハイフンや末尾のハイフンが残ったりする可能性があります。
- **提案された解決策:**
    - 堅牢な `sanitizeTaskName` ユーティリティまたはメソッドを導入する。
    - タスク名を安全な長さ（例：50〜100文字）に切り詰める。
    - 切り詰めが発生した場合、短いハッシュまたはタイムスタンプを付与してブランチ名の一意性を確保する。
    - 文字の置換を改善し、より多くのエッジケース（例：複数のハイフン、末尾のハイフン）を処理する。
- **受け入れ基準:**
    - タスク名が設定可能または安全な長さ（例：最大64文字）に切り詰められていること。
    - Gitブランチ名に無効なすべての文字が単一のハイフンに置き換えられていること。
    - 最終的な `safeTaskId` に先頭または末尾のハイフンが存在しないこと。
    - 生成された `environmentName` (例: `kanon-task-...`) が有効なGitブランチ名であること。
    - 長い名前、特殊文字、空の名前、ユニコード名など、さまざまなエッジケースをカバーするユニットテストが存在すること。
