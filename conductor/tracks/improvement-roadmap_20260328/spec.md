# 概要
本トラックは、Kanonの次期改善に向けた短期的なロードマップを作成することを目的とする。直近の比較分析から得られた洞察（takt等との比較）をもとに、主要な機能強化案を整理し、具体的な実装マイルストーンとして定義する。

# 機能要件 (ロードマップの記載内容)
ロードマップには以下の4つの改善領域を含め、直近（数週間〜1ヶ月程度）で着手・実現可能なフェーズ分けを行う。

1. **ワークフローの強化**: 
   - 既存のシーケンシャルな `pipeline.json` から、より柔軟で条件分岐が可能なPiece/Movement構造への移行計画。
2. **ループ監視機能 (AIによる進捗判定)**:
   - レビューと修正のループに対する、LLMを利用したスタック判定・強制終了メカニズムの導入計画。
3. **プロンプトのモジュール化 (Faceted Prompting)**:
   - エージェントのプロンプトをPersona, Policy, Knowledge, Instructionなどに分割し、再利用性を高めるためのアーキテクチャ設計計画。
4. **合議制の導入**:
   - 複数の観点からの並列レビューと多数決による判定システムの導入計画。

# 非機能要件
- ロードマップはプロジェクト内のMarkdownファイルとして出力し、段階的なマイルストーン（Phase 1, Phase 2...）形式で記載すること。
- 各マイルストーンには、想定されるタスクと期待される成果物を明記すること。

# 受け入れ基準
- 上記4つの領域すべてを網羅したロードマップドキュメントが作成されていること。
- ロードマップが短期的なマイルストーン形式（Markdown）になっていること。

# スコープ外
- 本トラック内での、ロードマップに記載された各機能の実際の実装・コーディング作業。
