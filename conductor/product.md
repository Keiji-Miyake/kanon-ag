# 初期コンセプト

Kanonは、複数のAIエージェント（Gemini, OpenCode-ai, GitHub Copilotなど）を統合し、マルチエージェント有限状態マシン（FSM）パイプラインを通じて、計画からコードレビューまでのソフトウェア開発ライフサイクル全体を自律的に推進するオーケストレーションツールです。

# プロジェクト概要

Kanonは、AIコーディングアシスタントのための「スーパーオーケストレーター」として機能します。個々のAI CLIツールと、状態、コンテキスト、フィードバックループを管理するまとまりのある開発ワークフローの間の架け橋となります。その主な目標は、人間の介入を最小限に抑えながら、AIエージェントが複雑なソフトウェアタスクで共同作業できる、信頼性の高い自動化された環境を提供することです。

# ターゲットユーザー

- **ソフトウェアエンジニア:** 定型的なコーディングタスクを自動化し、マルチエージェントのコラボレーションを活用したいと考えている方。
- **テックリード/アーキテクト:** 標準化されたAI駆動の開発ワークフローを実装し、エージェントのパフォーマンスを監視したいと考えている方。
- **DevOpsエンジニア:** 自律型AIエージェントをCI/CDパイプラインに統合し、自動化されたPRレビューや修正を行いたいと考えている方。

# 主な機能

- **マルチエージェント・オーケストレーション:** 異なるAI CLIに専門的な役割（アーキテクト、デベロッパー、レビュアー）を割り当てる柔軟なパイプライン。
- **チャット駆動型ワークフロー:** VS Code/Antigravityチャットを通じてエージェントを起動。実行前にエージェントと仕様やタスクを「壁打ち」して固めるプロセスを導入。
- **自律型ゲートキーパー:** 生成されたコードを自動的に検証（lint/ビルド）し、反復的な改善のためにエージェントにフィードバックを提供する自己修正ループ。
- **インタラクティブ・監視ダッシュボード:** Antigravityダッシュボードからエージェントの動きをリアルタイム監視。ユーザーが途中で口出し、割り込み、または停止（コスト超過防止）できる制御機能。
- **クリーンアーキテクチャとFSM:** スケーラビリティ、テスト可能性、および複雑なエージェント相互作用のための明確な状態管理を保証する堅牢な内部構造。
- **拡張可能なスキルシステム:** エージェントが新しい能力や専門知識を習得できるようにするモジュール式の「スキル」定義。

# コアバリュー

- **自律性:** エージェントがエラーを解決し、実装を独自に洗練できるようにすることで、人間の労力を削減します。
- **透明性:** ダッシュボードを通じて、エージェントの思考、行動、状態の変化を明確に可視化します。
- **誠実性:** 厳格な自動検証とプロジェクトの規約への準拠を通じて、高品質な出力を保証します。
