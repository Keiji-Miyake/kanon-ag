# Google JavaScript スタイルガイドの要約

このドキュメントでは、Google JavaScript スタイルガイドの主要なルールとベストプラクティスを要約しています。

## 1. ソースファイルの基本
- **ファイル命名:** すべて小文字で、アンダースコア (`_`) またはダッシュ (`-`) を使用します。拡張子は `.js` でなければなりません。
- **ファイルエンコーディング:** UTF-8。
- **空白:** ASCII 水平スペース (0x20) のみを使用してください。インデントにタブを使用することは禁止されています。

## 2. ソースファイルの構造
- 新しいファイルは ES モジュール (`import`/`export`) である必要があります。
- **エクスポート:** 名前付きエクスポートを使用してください (`export {MyClass};`)。**デフォルトエクスポートは使用しないでください。**
- **インポート:** 行をまたぐインポートは使用しないでください。インポートパスの `.js` 拡張子は必須です。

## 3. フォーマット
- **中括弧:** 1行のブロックであっても、すべての制御構造 (`if`, `for`, `while` など) に必要です。K&R スタイル ("Egyptian brackets") を使用してください。
- **インデント:** 新しいブロックごとに +2 スペース。
- **セミコロン:** すべてのステートメントはセミコロンで終了する必要があります。
- **列制限:** 80文字。
- **改行:** 継続行は少なくとも +4 スペースインデントします。
- **空白:** メソッド間には1行の空行を入れます。末尾の空白は禁止です。

## 4. 言語機能
- **変数宣言:** デフォルトでは `const` を使用し、再代入が必要な場合は `let` を使用します。**`var` は禁止されています。**
- **配列リテラル:** 末尾のカンマを使用してください。`Array` コンストラクタは使用しないでください。
- **オブジェクトリテラル:** 末尾のカンマとプロパティの短縮表記を使用してください。`Object` コンストラクタは使用しないでください。
- **クラス:** JavaScript のゲッター/セッタープロパティ (`get name()`) は使用しないでください。代わりに通常のメソッドを提供してください。
- **関数:** `this` コンテキストを維持するために、入れ子になった関数にはアロー関数を優先してください。
- **文字列リテラル:** シングルクォート (`'`) を使用してください。複数行の文字列や複雑な補完にはテンプレートリテラル (`` ` ``) を使用してください。
- **制御構造:** `for-of` ループを優先してください。`for-in` ループは、辞書形式のオブジェクトに対してのみ使用する必要があります。
- **`this`:** `this` は、クラスのコンストラクタ、メソッド、またはその中で定義されたアロー関数内でのみ使用してください。
- **等価チェック:** 常に厳密等価演算子 (`===` / `!==`) を使用してください。

## 5. 禁止されている機能
- `with` キーワード。
- `eval()` または `Function(...string)`。
- セミコロンの自動挿入。
- 組み込みオブジェクトの変更 (`Array.prototype.foo = ...`)。

## 6. 命名
- **クラス:** `UpperCamelCase` (アッパーキャメルケース)。
- **メソッドと関数:** `lowerCamelCase` (ローワーキャメルケース)。
- **定数:** `CONSTANT_CASE` (すべて大文字でアンダースコア区切り)。
- **定数以外のフィールドと変数:** `lowerCamelCase` (ローワーキャメルケース)。

## 7. JSDoc
- JSDoc は、すべてのクラス、フィールド、およびメソッドに使用されます。
- `@param`, `@return`, `@override`, `@deprecated` を使用してください。
- 型注釈は中括弧で囲みます (例: `/** @param {string} userName */`)。

*ソース: [Google JavaScript Style Guide](https://google.github.io/styleguide/jsguide.html)*
