# Google TypeScript スタイルガイドの要約

このドキュメントでは、`gts` ツールによって適用される Google TypeScript スタイルガイドの主要なルールとベストプラクティスを要約しています。

## 1. 言語機能
- **変数宣言:** 常に `const` または `let` を使用してください。**`var` は禁止されています。** デフォルトでは `const` を使用します。
- **モジュール:** ES6 モジュール (`import`/`export`) を使用してください。**`namespace` は使用しないでください。**
- **エクスポート:** 名前付きエクスポートを使用してください (`export {MyClass};`)。**デフォルトエクスポートは使用しないでください。**
- **クラス:**
  - **`#private` フィールドは使用しないでください。** TypeScript の `private` 可視性修飾子を使用してください。
  - コンストラクタ以外で再代入されないプロパティには `readonly` をマークしてください。
  - **`public` 修飾子は決して使用しないでください** (デフォルトです)。可能な限り、`private` または `protected` で可視性を制限してください。
- **関数:** 名前付き関数には関数宣言を優先してください。名無しの関数/コールバックにはアロー関数を使用してください。
- **文字列リテラル:** シングルクォート (`'`) を使用してください。補完や複数行の文字列にはテンプレートリテラル (`` ` ``) を使用してください。
- **等価チェック:** 常にトリプルイコール (`===`) と不等号 (`!==`) を使用してください。
- **型アサーション:** **型アサーション (`x as SomeType`) や非 null アサーション (`y!`) は避けてください**。どうしても使用する必要がある場合は、明確な正当性を示してください。

## 2. 禁止されている機能
- **`any` 型:** **`any` は避けてください**。`unknown` またはより具体的な型を優先してください。
- **ラッパーオブジェクト:** `String`, `Boolean`, `Number` のラッパークラスをインスタンス化しないでください。
- **セミコロンの自動挿入 (ASI):** これに依存しないでください。**すべてのステートメントを明示的にセミコロンで終了してください。**
- **`const enum`:** `const enum` は使用しないでください。代わりに通常の `enum` を使用してください。
- **`eval()` および `Function(...string)`:** 禁止。

## 3. 命名
- **`UpperCamelCase`:** クラス、インターフェース、型、列挙型、およびデコレータ。
- **`lowerCamelCase`:** 変数、パラメータ、関数、メソッド、およびプロパティ。
- **`CONSTANT_CASE`:** 列挙型の値を含むグローバルな定数値。
- **`_` 接頭辞/接尾辞:** プライベートプロパティを含め、識別子に **`_` を接頭辞または接尾辞として使用しないでください**。

## 4. 型システム
- **型推論:** 単純で明白な型については型推論に頼ってください。複雑な型については明示的に記述してください。
- **`undefined` と `null`:** 両方ともサポートされています。プロジェクト内で一貫性を保ってください。
- **オプション vs `|undefined`:** 型に `|undefined` を追加するよりも、オプションのパラメータやフィールド (`?`) を優先してください。
- **`Array<T>` 型:** 単純な型には `T[]` を使用します。より複雑な共用体型には `Array<T>` を使用します (例: `Array<string | number>`)。
- **`{}` 型:** **`{}` は使用しないでください**。`unknown`, `Record<string, unknown>`, または `object` を優先してください。

## 5. コメントとドキュメント
- **JSDoc:** ドキュメントには `/** JSDoc */` を使用し、実装のコメントには `//` を使用してください。
- **冗長性:** **`@param` や `@return` ブロックで型を宣言しないでください** (例: `/** @param {string} user */`)。これは TypeScript では冗長です。
- **情報の追加:** コメントは単にコードを言い換えるのではなく、情報を追加する必要があります。
