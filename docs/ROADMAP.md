# Kanon Improvement Roadmap (Short-term)

本ドキュメントは、Kanonプロジェクトの自律性と柔軟性を高めるための、直近数週間〜1ヶ月程度の改善ロードマップを定義します。

## 改善の4つの柱
1. **ワークフローの柔軟化 (Movement/Piece)**
2. **インテリジェントなループ監視 (Loop Monitor)**
3. **プロンプトのモジュール化 (Faceted Prompting)**
4. **合議制の導入 (Consensus/MAGI)**

---

## マイルストーン 1: 基礎構造の拡張 (Phase 1)
**目標**: 条件分岐が可能なワークフローの基礎と、ループ監視のプロトタイプを導入する。

### タスク
- [ ] **Workflow Rules**: `pipeline.json` に `rules` フィールドを追加し、エージェントの出力に基づく分岐をサポートする（**現在実装中**）。
- [x] **Loop Watchdog**: Review-Fixループにおいて、3回以上のリピートが発生した際に警告を出す単純なウォッチドッグの実装。

## マイルストーン 2: 自律性の向上 (Phase 2)
**目標**: AIによる進捗判定と、プロンプトの再利用性を高める。

### タスク
- [ ] **AI Judge**: ループ停滞時に、独立したエージェント（Supervisor）を起動し、ABORT または継続を判断する（**詳細設計完了: `docs/dev/design-ai-judge.md`**）。
- [ ] **Faceted Prompting (Alpha)**: エージェント定義から `policy` と `knowledge` を分離し、共通規約を読み込めるようにする（**基盤実装済み**）。

## マイルストーン 3: 信頼性と高度な連携 (Phase 3)
**目標**: 複数の視点による合議システムと、出力形式の厳格化を行う。

### タスク
- [x] **Parallel Review & Consensus**: 基礎ロジックの実装完了。
- [x] **Output Contract**: `OutputValidator` の基礎実装とモデル定義完了。
- [ ] **Advanced Validation**: `ajv` 等を用いた JSON Schema の厳格な検証の導入（**実装完了・レビュー指摘修正中**）。

## 今後の重要課題: エージェント実行の安定性向上
### 完了した改善
- [x] **インテリジェント・タイムアウト**: 無通信（Idle）状態を検知するタイムアウトを導入。
- [x] **リソース・バッファ管理**: 大規模出力を安全に処理するためのメモリ制御。
- [x] **自動リトライトリガーの精緻化**: 429 エラー等に対する高度なリトライ。
- [x] **AI Watchdog**: LLM によるループ停滞判定と回避策の提案。
- [x] **Self-Evolution Loop**: 自己改善（Score修正）および実行後リフレクション機能の実装。

### 残りの検討事項
- [ ] **状態復旧 (Checkpointing)**: 中断された Passage から、ワークツリーの状態を維持したまま再開（**詳細設計完了: `docs/dev/design-checkpointing.md`**）。

---

## 期待される成果
- **開発効率**: 無駄なループが早期に終了し、トークンコストと待ち時間を削減できる。
- **品質**: 複数の視点によるレビューと厳格な出力検証により、実装の品質が安定する。
- **拡張性**: 新しいワークフローやエージェントの役割を、YAMLベースで簡単に追加・再利用できるようになる。
