# Kanon Improvement Roadmap (Short-term)

本ドキュメントは、Kanonプロジェクトの自律性と柔軟性を高めるための、直近数週間〜1ヶ月程度の改善ロードマップを定義します。

## 改善の4つの柱
1. **ワークフローの柔軟化 (Movement/Piece)**
2. **インテリジェントなループ監視 (Loop Monitor)**
3. **プロンプトのモジュール化 (Faceted Prompting)**
4. **合議制の導入 (Consensus/MAGI)**

---

## マイルストーン 1: 基礎構造の拡張 (Phase 1)
**目標**: 条件分岐が可能なワークフローの基礎と、ループ監視のプロトタイプを導入する。

### タスク
- [ ] **Workflow Rules**: `pipeline.json` に `rules` フィールドを追加し、エージェントの出力（特定のキーワードや判定）に基づいた分岐をサポートする。
- [ ] **Loop Watchdog**: Review-Fixループにおいて、3回以上のリピートが発生した際に警告を出す単純なウォッチドッグの実装。

## マイルストーン 2: 自律性の向上 (Phase 2)
**目標**: AIによる進捗判定と、プロンプトの再利用性を高める。

### タスク
- [ ] **AI Judge**: ループ停滞時に、独立したエージェント（Supervisor）を起動し、コンテキストから進捗の有無を判定して ABORT または継続を判断する仕組みを導入。
- [ ] **Faceted Prompting (Alpha)**: エージェント定義から `policy` (コーディング規約) と `knowledge` を分離し、複数のフェーズで共通の規約を読み込めるようにする。

## マイルストーン 3: 信頼性と高度な連携 (Phase 3)
**目標**: 複数の視点による合議システムと、出力形式の厳格化を行う。

### タスク
- [x] **Parallel Review & Consensus**: 基礎ロジックの実装完了。
- [x] **Output Contract**: `OutputValidator` の基礎実装とモデル定義完了。
- [ ] **Advanced Validation**: `ajv` 等を用いた JSON Schema の厳格な検証の導入。

## 今後の重要課題: エージェント実行の安定性向上
**背景**: 自律開発（ドッグフーディング）の過程で、大規模なタスク実行時におけるタイムアウトや、エージェントプロセスのハング、レートリミットによる中断が課題として浮上した。

### 検討事項
- [ ] **インテリジェント・タイムアウト**: 単純な時間監視ではなく、ストリームの進捗に応じた動的なタイムアウト制御。
- [ ] **リソース・バッファ管理**: 大規模出力（10MB超）を安全に処理するためのメモリ・パイプ制御の最適化。
- [ ] **自動リトライトリガーの精緻化**: 429 (Rate Limit) 等の特定エラーに対する、より賢いバックオフ戦略の導入。
- [ ] **状態復旧 (Checkpointing)**: 中断された Passage から、ワークツリーの状態を維持したまま再開できる仕組みの強化。

---

## 期待される成果
- **開発効率**: 無駄なループが早期に終了し、トークンコストと待ち時間を削減できる。
- **品質**: 複数の視点によるレビューと厳格な出力検証により、実装の品質が安定する。
- **拡張性**: 新しいワークフローやエージェントの役割を、YAMLベースで簡単に追加・再利用できるようになる。
