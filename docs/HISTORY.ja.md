# 📜 プロジェクト移行履歴: kanon-ag への統合と深化

## 🗓️ 2026年2月14日 - プロジェクト名の変更とリブランディング

### 背景

本プロジェクトは、当初「AIエージェント向けのスキル集」としてスタートしましたが、開発が進むにつれて「エージェントチームを指揮するオーケストレーター」としての機能（Antigravity / Orchestrator）が中心的な役割を果たすようになりました。
より力強く、かつ汎用的な開発ツールとしてのアイデンティティを確立するため、プロジェクト名を **`kanon-ag`** へ変更し、コマンドラインツール名を **`kanon`** としました。

### 主要な変更点

1. **プロジェクト名の統一**:
    - `package.json` の `name` を `kanon-ag` へ変更。
    - 各種ドキュメント（README, ARCHITECTURE等）内の表記を一括更新。
    - リポジトリURLの参照先を `https://github.com/Keiji-Miyake/kanon-ag.git` へ更新。

2. **コマンド体系の刷新**:
    - `kanon lead`: 自律開発モード（Antigravityサーバー＋ダッシュボード）の起動。
    - `kanon dashboard`: ダッシュボード単体の起動。
    - `kanon init`: プロジェクトへの設定ファイル（`pipeline.json`等）の配備。

3. **ドキュメントの再編**:
    - はじめてのユーザー向けに **[GET_STARTED.ja.md](./GET_STARTED.ja.md)** を作成。
    - `README.ja.md` を「次世代オーケストレーション・ツール」としての魅力を伝えるデザインへ刷新。
    - `AGENTS.md` にオーケストレーター環境下での指示を追加。

4. **技術的改善**:
    - サーバー起動時のポート衝突回避（環境変数 `PORT` 対応）。
    - ログ出力を伴うサーバープロセスの正常起動を確認（`kanon lead` の安定化）。

### プロジェクトの進化

- 過去の全ての会話履歴およびコンテキスト情報は、AIエージェントによって維持されています。
- 実装された並列実行機能およびメモリバンク機能は、新生 `kanon-ag` の核として活用されます。
