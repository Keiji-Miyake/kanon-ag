# Antigravity マルチエージェント・チャットUI設計構想

## 概要

次回開発フェーズ（orchestration worktreeでの現在の作業完了後）にて実装を予定している、Antigravityエディタの全画面を活用したチャットUIの設計構想です。
各サブエージェント（Conductor, Developer, Tester, Architect, CI/CDなど）が自律的にやり取りする様子を、人間が俯瞰して見られるようにするためのマルチペインレイアウトを提供します。

## UIレイアウト（ワイヤーフレームに基づく要件）

全体の画面を垂直（またはグリッド）に分割し、各エージェント専用のチャットカラム（ペイン）を並べて表示する構成とします。

- **グローバル要素**
  - **ヘッダー**: `Antigravity` のタイトルロゴやグローバルなステータス表示
  - **サイドバー（左側）**: 既存のナビゲーションやツール切替、設定へのアクセス

- **メイン画面領域（マルチペイン/マルチカラム）**
  - **Conductor（指揮官）**: 全体の進行管理やユーザーへの最終報告を行うメインカラム。
  - **Developer（実装担当）**: 実際のコード生成やシステム上の変更作業ログが流れるカラム。
  - **Tester（テスト担当）**: テスト実行結果や検証のフィードバックストリーム。
  - **Architect / CI/CD**: 構成設計の議論やCI環境のビルド・デプロイログを表示するカラム。レイアウトによっては縦に分割するか、隣接するカラムとして配置。

## 機能要件

1. **ストリーミングなチャットログ表示**
   - 各エージェントから出力される思考プロセス（`thought`）や実行結果（`output`）が、それぞれの自身のカラムにリアルタイムにチャット形式で流れること。
   - 人間が見て「AI同士が協調して開発を進めている」ことが可視化できる体験の提供。
2. **ログの同期と視認性**
   - 誰が誰に指示を出したのか、フィードバックを返したのかが分かりやすいようなUI的工夫（バッジ、アイコン、時間軸によるスクロール位置の同期など）。
3. **エディタ画面の最大活用**
   - IDEやテキストエディタの横幅をフルに活用したレイアウト設計。
