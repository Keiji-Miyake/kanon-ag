# TEST_PLAN.md - スネークゲーム検証計画

## Scope
`games/snake/` 配下のHTML/CSS/JSによるスネークゲームの動作確認。

## Critical Paths
- ゲームの開始、移動、エサの取得、ゲームオーバー、リターンのサイクルが正しく機能すること。

## Test Cases

### 1. Basic (正常系)
- [ ] ページロード時にゲームが初期化され、ヘビが描画されているか。
- [ ] キーボード操作（矢印キー、WASD）でヘビが4方向に曲がるか。
- [ ] エサに接触するとスコアが10加算され、ヘビの体が1ブロック増えるか。
- [ ] スコアがハイスコアを超えた場合、LocalStorageに保存され更新されるか。

### 2. Edge (境界値・特殊系)
- [ ] 壁に衝突した瞬間にゲームが停止し、ゲームオーバー画面が表示されるか。
- [ ] 自分自身の体に衝突した場合にゲームオーバーになるか。
- [ ] 進行方向と真逆のキーを押した際、無視されるか（即座に自分自身に衝突するのを防ぐ）。
- [ ] エサがヘビの体と重ならない場所に再生成されるか。

### 3. Error (異常系)
- [ ] キーボード入力を高速に連打しても、パニックにならず入力がキューイングまたはフィルタリングされるか。（1フレーム1ターンの原則）
- [ ] ブラウザのサイズを変更しても、Canvasの描画が崩れないか。

## Verification Strategy
- 目視による動作確認。
- コンソールログでのエラーチェック。
