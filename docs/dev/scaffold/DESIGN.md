## 独立したスキルの原則
各スキルは他のスキルに依存せず、独立して機能する。エージェントが状況を判断し、適切なスキルを選択して活用する。

## 環境非依存の原則
OSや特定のユーザー環境に依存する絶対パスや設定を排除し、プロジェクトルートからの相対パスまたはプレースホルダーを用いた抽象的な記述を徹底する。
これにより、異なる開発環境間でのポータビリティを確保する。

## 主要コンポーネント

### 1. AGENTS.md (Single Source of Truth)
エージェント定義とプロジェクト全体の指示を統合。エージェントはこのファイルを読み込むことでコンテキストを把握する。

### 2. エージェントによる活用
エージェント（メインエージェントまたはサブエージェント）が、環境構築のタスクを遂行するために `scaffold` スキルの知識セットを使用する。特定のスキルが別の特定スキルを呼び出すことはない。

## 技術的決定
- **npx skills add の採用**: スキルのバイナリやファイルを直接コピーするのではなく、公式の管理コマンドを使用することで、最新性、依存関係の解決、AIモデルの選択を可能にする。
- **サブエージェント非依存**: スキル定義内ではエージェントの内部構成（サブエージェント等）には言及せず、呼び出し側のエージェントがその実行戦略を決定する。

## データフロー
1. `dev-support` が `AGENTS.md` の不在を確認。
2. `scaffold` スキルを適用。
3. 手順に従い `AGENTS.md` を作成。
4. 必要なスキルを `npx skills add` で導入。
5. 構築完了。
