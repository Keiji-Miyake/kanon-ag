# Human-in-the-Loop (HITL) 動作テスト手順

このドキュメントでは、実装したリアルタイム監視、介入（口出し）、および停止機能の動作確認手順を説明します。

---

## 1. ターミナル (CLI) でのテスト

ターミナルのダッシュボードのみを使用して機能を検証する手順です。

### 手順
1. **エージェントの起動 (Terminal 1)**
   少し時間のかかるタスクを指定して、Kanonを実行します。
   ```bash
   kanon execute --task "demo/hitl-test.ts にフィボナッチ数列を計算する関数を追加し、そのテストも作成してください。再帰を使って実装してください。"
   ```

2. **ダッシュボードの起動 (Terminal 2)**
   別のターミナルで、リアルタイム監視・介入用ダッシュボードを起動します。
   ```bash
   kanon dashboard
   ```

3. **介入 (口出し) の実行**
   ダッシュボードで `i` キーを押し、メッセージを入力して Enter を押します。
   * **例:** 「再帰ではなくループ（反復処理）を使って実装し直してください。」
   * **期待される結果:** Terminal 1 の出力に `[User Intervention]` が表示され、エージェントが指示に従って方針を修正すること。

4. **停止 (Stop) の実行**
   エージェントが動作中に `s` キーを押します。
   * **期待される結果:** CLIプロセスが直ちに終了し、関連する子プロセス（sleep等）もすべてクリーンアップされること。

## 2. VS Code 拡張機能 UI でのテスト

VS Code のサイドバー UI と CLI の連携を検証する手順です。

### 手順
1. **拡張機能のデバッグ起動**
   * VS Codeで `kanon-ag` プロジェクトを開く。
   * `F5` キーを押し、「Extension Development Host」ウィンドウを起動する。

2. **ダッシュボードを開く**
   * 新しいウィンドウのサイドバーにある **Kanonアイコン** をクリックしてダッシュボードを表示する。
   * ステータスが `disconnected` であることを確認。

3. **エージェントの起動 (Terminal)**
   メインウィンドウまたはシステムのターミナルから、CLIを起動します。
   ```bash
   kanon execute --task "demo/hitl-test.ts に複雑な数値計算ロジックを追加し、丁寧にコメントを付けてください。"
   ```

4. **UI からの介入**
   * サイドバーのステータスが `connected` になり、ログが流れ始めることを確認。
   * UIの入力欄に「コメントはすべて日本語で書いてください」と入力し、「送信」をクリック。
   * **期待される結果:** エージェントがその後のコード生成において指示（日本語コメント）を反映すること。

5. **UI からの停止**
   * サイドバーの赤い **「🛑 Stop」** ボタンをクリック。
   * **期待される結果:** 実行中のターミナルのCLIプロセスが即座に停止すること。

---

## 検証のポイント
- [ ] 介入メッセージがプロンプトの `[USER INTERVENTION]` セクションに正しく挿入されているか。
- [ ] 停止ボタンによって、子プロセス（特にシェル経由で実行されたコマンド）がゾンビにならずに終了しているか。
- [ ] ネットワーク切断時やタイムアウト時のエラーハンドリングが適切か。
