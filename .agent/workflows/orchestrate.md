# /orchestrate - 自律開発パイプライン

ユーザーからタスクを受け取り、Antigravity (Conductor) として振る舞う `kanon` CLI が、全行程を自律的に進行させます。

// turbo-all

---

## 0. タスクの確認

ユーザーにタスクの内容を確認します。まだタスクが指定されていない場合は、ユーザーに入力を求めてください。

タスクが確定したら、以下の情報を収集してください：

- タスク内容（何を作るか / 何を修正するか）

---

## 1. Antigravity Global Orchestration

`kanon run` コマンドを実行して、計画(Architect)・実装(Developer)・レビュー(Reviewer) の全工程を一括で実行します。
Zero Approval対応済みのため、ユーザーの承認や追加指示は不要です。完了まで自律的に動作します。

// turbo

```bash
kanon run --task="{{TASK}}"
```

---

## 2. 最終成果物の動作確認

`kanon run` コマンドが完了したら、生成されたアプリケーションまたはモジュールが**実際に正しく動作するか**を確認してください。

- 動作確認としてビルドコマンドや実行コマンド（例: `npm run start`, `node index.js` 等）をターミナルで実行します。
- **🚨 重要:** 動作確認のコマンドを実行する際は、ユーザーの承認待ち（Runボタンのクリック）を発生させないよう、必ず自動実行仕様（`SafeToAutoRun: true` や `// turbo` など）でコマンドを呼び出してください。完全自律で検証を終える必要があります。
- 動作に不具合がある場合は、追加で修正作業を行ってください。

---

## 3. 完了報告書の作成 (walkthrough.md)

動作確認が完了したら、ユーザー向けに**完了報告書（`walkthrough.md`）**を作成し、プロジェクトのルートディレクトリに出力してください。
報告書には以下の内容を含めます：

- 何を作成（実装）したか
- どのように実行・テストするか
- ユーザーへの最終確認事項

---

## 完了報告

Dashboardのログ、動作確認の結果、および `walkthrough.md` の出力をユーザーに報告し、タスクを完了してください。
