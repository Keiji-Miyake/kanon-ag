# /orchestrate - 自律開発パイプライン

ユーザーからタスクを受け取り、Antigravity (Conductor) として振る舞う `kanon` CLI が、全行程を対話的に進行させます。

---

## 0. タスクの確認・相談 (User -> Conductor)

まずはユーザーがやりたいこと、抱えている課題や悩みをヒアリングし、深く理解するように努めてください。
単なるタスクの確認にとどまらず、**ユーザーの考えがまとまるまで対話（壁打ち）を行ってください。**

**具体的な進め方:**
1. ユーザーの入力を元に、**実装すべき仕様の概要**と**TODOリスト（やるべきこと）**を提案してください。
2. ユーザーからのフィードバックを受けて、内容をブラッシュアップしてください。
3. 最終的にどのような結果（アウトプットやプラン）が欲しいかを明確に定義します。

タスクの目的と期待する結果について双方の合意が取れたら、次のステップに進むための**明確な選択肢をユーザーに提示してください**。
（例）

1. **作業を開始する**（提示した仕様とTODOを元に `kanon plan` を実行する）
2. 仕様を修正する / さらに深掘りする
3. その他（コメント等）

ユーザーが「1」や「進めて」などと応え、明確な実行の承認（OK）を得てから初めて次のステップへ進んでください。


---

## 1. 設計フェーズ (Plan)

ユーザーの承認後、`kanon plan` コマンドを実行して Architect エージェントに設計図（`implementation_plan.md`）の作成を依頼します。

// turbo

```bash
kanon plan --task="{{TASK}}"
```

設計が完了したら、生成された `implementation_plan.md` の内容（または実行結果）を読み取り、簡潔にユーザーへ説明してください。
**必ずユーザーに「実装フェーズ（Go）に進んでよいか？」を確認**し、承認を得てから次のステップへ進んでください。

---

## 2. 実装・検証フェーズ (Go)

ユーザーの実装指示（Go）後、`kanon execute` コマンドを実行し、Developer, Creator, Reviewer, Tester と連携して実装および自己修正ループを実行します。

// turbo

```bash
kanon execute --task="{{TASK}}"
```

---

## 3. 最終成果物の動作確認 (Tester)

`kanon execute` コマンドが完了したら、生成されたアプリケーションまたはモジュールが**実際に正しく動作するか**を確認してください。

- 動作確認としてビルドコマンドや実行コマンド（例: `npm run start`, `npm test` 等）をターミナルで実行します。
- **🚨 重要:** 動作確認のコマンドを実行する際は、ユーザーの承認待ち（Runボタンのクリック）を発生させないよう、必ず自動実行仕様（`SafeToAutoRun: true` や `// turbo` など）でコマンドを呼び出してください。
- 動作に不具合がある場合は、必要に応じて追加で修正作業を行ってください。

---

## 4. 完了報告書の作成と最終報告 (Conductor -> User)

動作確認が完了したら、`kanon report` コマンドを実行して、ユーザー向けの**完了報告書（`walkthrough.md`）**を作成し、プロジェクトのルートディレクトリに出力してください。

// turbo

```bash
kanon report --task="{{TASK}}"
```

報告書には以下の内容を含めます：

- 何を作成（実装）したか
- どのように実行・テストするか
- ユーザーへの最終確認事項

最後に、Dashboardのログ、動作確認の結果、および `walkthrough.md` の出力をユーザーに報告し、**最終的なOK/NGのフィードバック**を求めてください。
NGの場合は追加の指示を元に再度修正を行い、OKであればタスクを完了してください。
