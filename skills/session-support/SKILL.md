# Session Support Skill

このスキルは、開発セッションの開始・終了を円滑にし、作業の文脈（- **コンテキストの保持**: セッションの中断時、現在の作業状態、未完了タスク、次のステップを作業ログ（Context）に記録し、再開時のロスをゼロにする。
- **構造の維持**: プロジェクトのドキュメント構造やディレクトリ構成が維持されているか監視し、秩序を保つ。
- **ナビゲーション**: 開発の各フェーズ（設計、実装、テスト）への移行を支援する。

## ワークフロー

### 1. セッション開始 (Resume/Start)
- **現状把握**:
   - 作業ログやロードマップを読み込み、前回の終了地点を確認。
   - 未完了のタスクを特定し、ユーザーに「何から始めるか」を確認。

### 2. 作業中の支援 (Support)
- **作業ログの更新**:
   - 重要な決定や進捗があった場合、リアルタイムで作業ログの更新を促す。

### 3. セッション終了 (Suspend)
- **終了処理**:
   - 変更内容をコミット（またはステージング）の状態確認。
   - 「次にやるべきこと」を明確にして作業ログに記録。
   - 必要に応じてセッション管理スクリプトを実行し、現状のステータスをドキュメントに記録する。
- **構造チェック**:
   - 新しい機能やドキュメントが適切に配置されているか確認。
5. **Git コミットの推奨**: WIP (Work In Progress) を含むコミットを促す。

## 構造の維持 (構造の番人)

以下の標準的なディレクトリ構造が守られているか確認し、逸脱があれば修正を提案してください：

```
docs/
├── ARCHITECTURE.md
└── dev/
    ├── README.md (機能インデックス)
    └── [feature-name]/
        ├── SPEC.md
        ├── DESIGN.md
        └── CONTEXT.md
```

## 注意事項

- このスキルは「プロジェクトの指揮（意思決定やロードマップ策定）」は担当しません。それらが必要な場合は `conductor` スキルを活用してください。
- ユーザーの好みを尊重しつつ、プロジェクトの健全性を維持するためのベストプラクティスを推奨してください。
